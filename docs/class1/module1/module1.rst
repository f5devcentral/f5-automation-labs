Module 1 – REST APIの基礎とデバイスのオンボーディング
==============================================

このモジュールでは、BIG-IP iControl REST APIと対話するために必要な基本的な概念を学習します。
また、BIG-IPの冗長構成（Active / Standby）を実現する典型的なデバイスのオンボーディング・ワークフローについても説明します。 
このモジュールは従来の**命令型**のアプローチを採用しています。 

.. NOTE::

このラボには2台のBIG-IPが含まれていますが、割り当てられた時間内に全項目を完了できるように、ほとんどのラボではBIG-IP-Aというデバイスのみの設定に重点を置いています。(BIG-IP-Aの管理IP設定とライセンス・アクティベーションは既に完了しており、BIG-IP-Bにもいくつかの最小構成がロードされています。) 実環境では、すべてのBIG-IPでデバイスオンボーディング用の初期設定を実行する必要があります。

.. NOTE::

このラボで実行されたAPIコマンドを確認するには、BIG-IPおよびiWorkflowデバイスに対してGUIまたはSSHセッションを開くことを推奨します。 また、トラブルシューティングや、エラー原因を特定するために以下のBIG-IPログを監視することもできます。

    - BIG-IP:

      - /var/log/ltm
      
      - /var/log/restjavad.0.log

    - iWorkflow: 

      - /var/log/restjavad.0.log
